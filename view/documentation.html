Templates Get the same look and feel across all your pages

Add a 'data-sw-view' attribute to a <div> tag, to specify where your views are inserted into your templates. 

<html>
  <head>
    <title>My Sample Site</title>
  </head>
  <body>
    <header>
      Placeholder for header component
    </header>
    <div data-sw-view="">
      Placedholder for view
    </div>
    <footer>
      Placedholder for footer component
    </footer>
  </body>
</html>

Views The epicentre of your site, organise your content

Out of the box a view will look for a template called "default". You can override this by providing an optional meta tag, see below.

The template title can be overriden by providing an optional title element in the same place. 

<html>
  <head
    <title>My Title</title>
    <meta name="template" content="example-template" />
  </head>
  <body>
    <div>
      <p>This is the content for the view<p>
    </div>
  </body>
</html>

Components Contain reusable mark-up

Simple components enable re-use of structural or semantic markup.

Component mark up must be placed into the components directory. Everything inside the <body> tag is injected into the caller. The component's name is its file-name minus extension.

An example of a simple component is given below. 

<html>
  <body>
    <span>
      My Sample Site
    </span>
  </body>
</html>

Call the component from a template, view or another component with the following mark-up. 

<span data-sw-component="site-name" />

Data driven components Populated via data files

Once configured a data driven component allows you to add contents to your site without editing the mark-up.

A source is a collection of like articles, for example news stories or blog posts. A source can be any directory under the data directory . The following component call will gather contents from data/news and inject it into components/news.html 

<span data-sw-component="my-news" data-sw-source="news" />

Source items are stored in .edn files. The following specifies what the title and description should be. 

{
  :title "Lorem ipsum"
  :description "Lorem ipsum dolor sit amet, consectetuer adipiscing elit, sed diam nonummy nibh euismod tincidunt ut laoreet dolore magna aliquam erat volutpat."
}

The component mark-up presents whatever data you want to extract from the edn files. Below we are extracting a title and a description from the data files. 

<html>
  <body>
    <ul>
      <li>
        <h2 data-sw-text="title">Placeholder for title</h2>
        <div data-sw-text="description">Placeholder for description</div>
      </li>
    </ul>
  </body>
</html>

Other data related attributes include:

  • data-sw-sort (chooses an item of data to sort on)
  • data-sw-sort-dir (specify ascending or descending)
  • data-sw-limit (limit the number of items in your list)

Details Page A dedicated page for a sourced component item

A details page is a dedicated template for rendering whole articles belonging to a specific source. If you have a datasource 'news' in data/news every article in it can be rendered in a detail template 'news' under /template/detail.

A sample details page is listed below. 

<html>
  <body>
    <div data-sw-component="detail/body">Placeholder for component</div>
  </body>
</html>

Please note only components can extract data from datasources, so your details page should include a component.

Link to a details page from a component using a special attribute value 'sw/path', demonstrated below. 

<html>
  <body>
    <a data-sw-href="sw/path" href="#">Comments</a>
  </body>
</html>

Type Specifies how a component should be rendered

The default display type is flat (Non-tree). The only other type is Tree. Tree components place HTML into sub elements based on your source hierarchy structure. E.G. Directory structure of .edn files. 

<div data-sw-component="my-tree" data-sw-source="my-source" data-sw-type="tree" />
data/
└── my-source/
    ├── sub/
    │   └── item1.edn
    ├── item1.edn
    └── item2.edn

Due to the unique nature of tree components. The mark-up of tree components can not be altered. Below is an example of the tree components markup to help style. 

<ul>
  <li class="folder">my-source
    <ul>
      <li class="folder">sub
        <ul>
          <li><a href="directory/item1.html">item1.edn</a></li>
        </ul>
      </li>
    </ul>
  </li>
  <li><a href="directory/item1.html">item1.edn</a></li>
  <li><a href="directory/item2.html">item2.edn</a></li>
</ul>

Please note a tree component requires a details component to be created.
