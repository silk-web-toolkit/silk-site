Templates Get the same look and feel across all your pages

Add a data-sw-view attribute to a <div> tag, to specify where your views are inserted into your templates. 

<html>
  <head>
    <title>My Sample Site</title>
  </head>
  <body>
    <header>
      Placeholder for header component
    </header>
    <div data-sw-view="">
      Placedholder for view
    </div>
    <footer>
      Placedholder for footer component
    </footer>
  </body>
</html>

Views The epicentre of your site, organise your content

Out of the box a view will look for a template called "default". You can override this by providing an optional <meta> tag, see below.

The template title can be overriden by providing an optional title element in the same place. 

<html>
  <head
    <title>My Title</title>
    <meta name="template" content="example-template" />
  </head>
  <body>
    <div>
      <p>This is the content for the view<p>
    </div>
  </body>
</html>

Components Contain reusable mark-up

Simple components enable re-use of structural or semantic mark-up.

Component mark up must be placed into the components directory. Everything inside the <body> tag is injected into the caller. The component's name is its file-name minus extension.

An example of a simple component is given below. 

<html>
  <body>
    <span>
      My Sample Site
    </span>
  </body>
</html>

Call the component from a template, view or another component with the following mark-up. 

<span data-sw-component="site-name" />

Data driven components Populated via data files

Once configured a data driven component allows you to add contents to your site without editing the mark-up.

A source is a collection of like articles, for example news stories or blog posts. A source can be any directory under the data directory . The following component call will gather contents from data/news and inject it into components/news.html 

<span data-sw-component="my-news" data-sw-source="news" />

Source items are stored in .edn files. The following specifies what the title and description should be. 

{
:title "Lorem ipsum"
:description "Lorem ipsum dolor sit amet, consectetuer adipiscing elit, sed diam nonummy nibh euismod tincidunt ut laoreet dolore magna aliquam erat volutpat."
}

The component mark-up presents whatever data you want to extract from the edn files. Below we are extracting a title and a description from the data files. 

<html>
<body>
  <ul>
    <li>
      <h2 data-sw-text="title">Placeholder for title</h2>
      <div data-sw-text="description">Placeholder for description</div>
    </li>
  </ul>
</body>
</html>

Other data related attributes include:

  • data-sw-sort (chooses an item of data to sort on)
  • data-sw-sort-dir (specify ascending or descending)
  • data-sw-limit (limit the number of items in your list)

Details Page A dedicated page for a sourced component item

A details page is a dedicated template for rendering whole articles belonging to a specific source. If you have a datasource 'news' in data/news every article in it can be rendered in a detail template 'news' under /template/detail.

A sample details page is listed below. 

<html>
<body>
  <div data-sw-component="detail/body">Placeholder for component</div>
</body>
</html>

Please note only components can extract data from datasources, so your details page should include a component.

Link to a details page from a component using a special attribute value sw/path, demonstrated below. 

<html>
<body>
  <a data-sw-href="sw/path" href="#">Comments</a>
</body>
</html>

Component Type Specifies how a component should be rendered

The default display type is flat (Non-tree). The only other type is Tree. Tree components place HTML into sub elements based on your source hierarchy structure. E.G. Directory structure of .edn files. 

<div data-sw-component="my-tree" data-sw-source="my-source" data-sw-type="tree" />
data/
└── my-source/
  ├── sub/
  │   └── item1.edn
  ├── item1.edn
  └── item2.edn

Due to the unique nature of tree components. The mark-up of tree components can not be altered. Below is an example of the tree components mark-up to help style. 

<ul>
<li class="folder">my-source
  <ul>
    <li class="folder">sub
      <ul>
        <li><a href="directory/item1.html">item1.edn</a></li>
      </ul>
    </li>
  </ul>
</li>
<li><a href="directory/item1.html">item1.edn</a></li>
<li><a href="directory/item2.html">item2.edn</a></li>
</ul>

Please note a tree component requires a details component to be created.

Navigation Generate site menu's from data driven components.

The offers two forms of navigation, views and bookmarks. View navigation caters for site hierarchy. Bookmark navigation for sections inside a view. If you would like to see the data generated for navigation look in the silk home directory ~/.silk/sw.

Data for view navigation is generated by including the data-sw-nav attribute in a view's body tag. Views can be sorted by the data-sw-priority attribute. Below is an example of how the index.html will typically look. 

<html>
  <body data-sw-nav="Home" data-sw-priority="1">
    <div>
      <p>This is the content for the view<p>
    </div>
  </body>
<html>

Data for page bookmarks is generated by placing the data-sw-nav attribute in the tag containing the linked anchor tag id, like so. 

<section data-sw-nav="Section 1" id="sec1">
  <h2>Section 1</h2>
  <p>
    This is the content for the section 1
  </p>
</section>
Please note it is not mandatory to use a section tag.

Views and bookmarks data are written into a component, to create a menu, by setting the data-sw-source attribute as sw/views or sw/bookmarks/index. The bookmarks should specify which view they are found on so in the previous example index is the view. You may also wish to sort the views by priority like so. 

<div data-sw-component="view-menu" data-sw-source="sw/views" data-sw-sort="priority" data-sw-sort-dir="ascending">
  Placeholder for header menu component
</div>

The mark-up for the view menu component must contain path and title to form data injected links. 

<html>
  <body>
    <ul>
      <li>
        <a data-sw-href="path">
          <span data-sw-text="title">Placeholder for view titles</span>
        </a>
      </li>
    </ul>
  </body>
</html>

The mark-up for the bookmark menu component must contain HTML's anchor tag # notation like so. 

<html>
  <body>
    <div>
      <ul>
        <li>
          <a data-sw-href="#path" data-sw-text="title">
            Placeholder for bookmark title
          </a>
        </li>
      </ul>
    </div>
  </body>
</html>

Tipue Search Support Control your searchable content dynamically

We recommend you look at Tipue Search, an open source JQuery search engine, along with our Blog example. This search engine has two modes "Live" and "JSON".

"Live" automatically indexes your site but due to the Same Origin Policy it requires your site to be on the web (live) for it to work.

"JSON" mode requires a JavaScript file with search terms to be maintained.

By adding attribute data-sw-content to your markup the will allow you to include a JavaScript file (resource/js/tipuesearch_content.js) which contains the "JSON" search terms. The data-sw-content attribute can be added to Templates, Views, Components and Details Pages. Navigation attributes are used to gather titles and links for your searchable contents.

Below is an example of how to include multiple searchable sections to a page with bookmark navigation. 

<html>
  <body data-sw-nav="Documentation" data-sw-priority="2" >
    <section data-sw-nav="Section 1" id="sec1" data-sw-content>
      <p>Some searchable content.</p>
      <p>Some more.</p>
    </section>
    <section data-sw-nav="Section 2" id="sec2" data-sw-content>
      <p>Some searchable content.</p>
      <p>Some more.</p>
    </section>
  </body>
</html>
