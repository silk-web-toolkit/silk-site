Templates Structure your site once & reuse it on each page

Add the following attribute 'data-sw-view', to <div> tag, to specify where your views are inserted the surround it with your site contents. 

<html>
<head>
  <title>My Sample Site</title>
</head>
<body>
  <header>
    Placeholder for header component
  </header>
  <div data-sw-view="">
    Placedholder for view
  </div>
  <footer>
    Placedholder for footer component
  </footer>
</body>
</html>

Views Wrap into templates and become a page on your site

This view points to a template called "example-template". If you leave the <meta> tag blank the will look for a template called "default". 

<html>
  <meta name="template" content="example-template" />
<body>
  <div>
    <p>This is the content for the view<p>
  </div>
</body>
</html>

Components Contain reusable mark-up

The most basic component you may wish to create for your site maybe one containing your site name. Doing so means you can control it one place.

Component mark up must be placed into the components directory. Everything inside the <body> tag is what is inserted. The component's name is its file-name minus extension. 

<html>
<body>
  <span>
    My Sample Site
  </span>
</body>
</html>

Call the component from a template, view or another component with the following mark-up. 

<span data-sw-component="site-name" />

Source A component attribute which specifies where to read content from

Once configure a sourced component allows you to add contents to your site without editing the mark-up.

A source can be a directory in your site where you wish to draw contents. The following component call will gather contents from data/news/ and inject it into components/news.html 

<span data-sw-component="my-news" data-sw-source="news" />

Source items are stored in .edn files. The following specifies what the title and description should be. 

{
  :title "Lorem ipsum"
  :description "Lorem ipsum dolor sit amet, consectetuer adipiscing elit, sed diam nonummy nibh euismod tincidunt ut laoreet dolore magna aliquam erat volutpat."
}

The component mark-up formats the source. 

<html>
<body>
  <ul>
    <li>
      <h2 data-sw-text="title">Placeholder for title</h2>
      <div data-sw-text="description">Placeholder for description</div>
    </li>
  </ul>
</body>
</html>

Details Page A dedicated page for a sourced component item

If a details page is detected (in the components/details with a directory with file-name matching the source) then a link can be placed in the sourced component like so. 

<html>
<body>
  <ul>
    <li>
      <h2 data-sw-text="title">Placeholder for title</h2>
      <div data-sw-text="description">Placeholder for description</div>
      <a data-sw-href="path" href="#">Further details</a>
    </li>
  </ul>
</body>
</html>

Further details can be added to the source.

{
  :title "Lorem ipsum" 
  :description "Lorem ipsum dolor sit amet, consectetuer adipiscing elit, sed diam nonummy nibh euismod tincidunt ut laoreet dolore magna aliquam erat volutpat."
  :further-details "Lorem ipsum dolor sit amet, consectetuer adipiscing elit, sed diam nonummy nibh euismod tincidunt ut laoreet dolore magna aliquam erat volutpat. 
  
Ut wisi enim ad minim veniam, quis nostrud exerci tation ullamcorper suscipit lobortis nisl ut aliquip ex ea commodo consequat."
}

The details component formats the source. 

<html>
<body>
  <ul>
    <li>
      <h2 data-sw-text="title">Placeholder for title</h2>
      <div data-sw-text="description">Placeholder for description</div>
    </li>
  </ul>
</body>
</html>

Type A component attribute which specifies the how to render

The default display type is flat (Non-tree). The only other type is Tree. Tree components place HTML into sub elements based on your source hierarchy structure. E.G. Directory structure of .edn files. 

<div data-sw-component="my-tree" data-sw-source="my-source" data-sw-type="tree" />
data/
└── my-source/
    ├── sub/
    │   └── item1.edn
    ├── item1.edn
    └── item2.edn

Due to the unique nature of tree components. The mark-up of tree components can not be altered. Below is an example of the tree components mark-up up to help style. 

<ul>
  <li class="folder">my-source
    <ul>
      <li class="folder">sub
        <ul>
          <li><a href="directory/item1.html">item1.edn</a></li>
        </ul>
      </li>
    </ul>
  </li>
  <li><a href="directory/item1.html">item1.edn</a></li>
  <li><a href="directory/item2.html">item2.edn</a></li>
</ul>

Please note a tree component requires a details component to be created.
