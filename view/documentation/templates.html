Documentation Templates

Most sites have at least one template, with design elements including headers, footers, navigation etc. This is where we start. See the sample Silk site for an example template.

Templates should contain a div element with id 'silk-view' if you want any views to be mixed in later. Apart from this requirement, any content can be placed inside the 'body' element, including multiple container elements.

We specify templates in our views, in the head element with meta tags. If no template is specified then a 'default.html' will be used which must exist in the templates directory.

Below is a simple example of a template.

<html>
  <head>
    <title>Sample template</title>
  </head>
  <body>
    <div>
      Some header content
    </div>
    <div id="silk-view">
      View content will be inserted here
    </div>
    <div>
      Some footer content
    </div>
  </body>
</html>
