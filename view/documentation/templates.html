Documentation Templates

Most sites have at least one template, with design elements including headers, footers, navigation etc. This is where we start.

With the default and most common pipeline templates are required to be valid xhtml and should contain a div element with id 'silk-view' if you want any views to be mixed in later. Apart from this requirement, any content can be placed inside the 'body' element, including multiple container elements.

If you have cloned a site prototype, and you are working with the default Silk pipeline there is a good chance that there is a default template prepared for you called 'default.html' under the 'template' directory. This template is automatically wrapped around any view which does not specify a particular template at spin time. If a given view does not specify a template, and the default.html template is not present in your project Silk will wrap a standard 'template-missing' template around your view.

Below is a simple example of a template.

<html>
  <head>
    <title>Sample template</title>
  </head>
  <body>
    <div>
      Some header content
    </div>
    <div id="silk-view">
      View content will be inserted here
    </div>
    <div>
      Some footer content
    </div>
  </body>
</html>

It is also possible as we mentioned above to specify which template should be wrapped around a view. This gives us greater flexibility in composing sites. If the template you specify does not exist locally, Silk will look at core templates, to see if the template exists there.
