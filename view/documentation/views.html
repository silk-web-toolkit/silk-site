Documentation Views

Without a view, or views, there is no website or webapp or set of web services. Views are the epicentre with Silk.

In Silk all views will render independently from the template... everything is designer friendly. Start with valid html and body elements, then ensure everything you want to be pulled in as the view is inside a div element with id 'silk-view'. 

  <html>
    <body>
      <div id="silk-view">
        <p>This is the content for the view<p>
      </div>
    </body>
  </html>
