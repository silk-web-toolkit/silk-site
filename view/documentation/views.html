Documentation Views

Without a view, or views, there is no website or webapp or set of web services. Views are the epicentre with Silk.

In Silk all views will render independently from the template... everything is designer friendly. Start with valid 'html' and 'body' elements, then it is best to have some container element as a root inside of the 'body' element. 

  <html>
    <body>
      <div>
        <p>This is the content for the view<p>
      </div>
    </body>
  </html>
