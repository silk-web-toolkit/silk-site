Documentation Components

Components provide a means to build sites out of reusable lego like blocks of content. See the sample Silk site for an example of a component. Both templates and views can call in components.

Anatomy of a component

Again components start with a valid 'html' and 'body' element. It is best then to have one top level container element inside the 'body' element. It is this content that will be injected where the component is invoked. Below is an example of a static component called 'sample'. 

<html>
  <body>
    <div>
      <p>Illustrates the markup for pulling in a simple static component.</p>
    </div>
  </body>
</html>

Types of component

There are two different types of component; local and shared.

Local
located in a 'components' directory local to the project
simple to manage
cannot be shared
Shared
Live in a separate location on your machine often managed by defining a SILK_COMPONENTS_PATH environment variable
managed with Twitter Bower
can be shared between individuals, teams, departments or companies, using servers, Dropbox etc
