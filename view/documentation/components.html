Documentation Components

Components provide a means to build sites out of reusable lego like blocks of content. Your Silk generated site may have an example of a component. Both templates and views can call in components.

Anatomy of a component

Again components start with a valid 'html' and 'body' element. It is best then to have one top level container element inside the 'body' element. It is this content that will be injected where the component is invoked. Below is an example of a static component called 'sample'. 

<html>
  <body>
    <div>
      <p>Illustrates the markup for pulling in a simple static component.</p>
    </div>
  </body>
</html>

Types of component

There are three different types of component; local, core (in a specified package) and core org.silkyweb (those shipped with Silk out of the box).

  • Local to the project (found in any Silk project under the 'components' directory) - always your components
  • Local repository within a specific package (in core silk) - may be your components, may be other companies'
  • Local repository within org.silkyweb (distributed with Silk)

To pull in a component in your own package in the local repository the id attribute might be something like: 'silk-component:com/mycompany/simple-address'.

The rules for component lookup are simple. If a component is not found locally, Silk will look for the specified path treating it as a package hierarchy in the local repository. If lookup fails again, the component is searched for under the default package 'org.silkyweb' in the local repository. Finally, if the requested component does not exist under the default package, the 'component-missing' component is inserted into your view or template.
