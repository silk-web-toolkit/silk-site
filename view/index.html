A better way to build the web.

Vision

Rich websites, webapps and webservices built with re-usable components using nothing but vanilla HTML5 and CSS by default. A well designed web publishing toolkit, with powerful technologies under the hood rolled into an incredibly intuitive, simple interface. Eradicate the battle with server side languages by just neatly side-stepping them. End semantic pollution... Silk is POSH (Plain Old Semantic HTML5).

Look at the change in just 10 years from a server side perspective versus the change in HTML over 20 years. Which would you choose to invest in ? Is there a bigger user community than the HTML user community ? It's time to let these users build with HTML !

How many of your websites really need to be dynamic beyond a contact form ? Every time you use a dynamic web framework you open attack vectors on what you are delivering to your clients. Silk works at compile time, there are fewer security issues.

Lower the barrier to entry for developers and designers on your team, anyone can work with HTML5 and CSS.
